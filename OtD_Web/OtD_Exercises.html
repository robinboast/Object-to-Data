    Install jq

    jq is small command line tool that can process JSON formatted data. Many APIs return their data in JSON format, a worldwide standard for exchanging data. With jq you can transform and extract information from JSON data. To install jq, there are different steps for Cygwin users and Unix/Linux/Mac users.

    Goals:

    Steps:

    1. Cygwin users should be able to install jq with the following three commands: 1) 'wget --no-check-certificate https://github.com/stedolan/jq/releases/download/jq-1.5/jq-1.5.tar.gz', 2) 'tar xzf jq-1.5.tar.gz' 3) 'cd jq-1.5; ./configure && make && cygstart --action=runas make install'. This will then download and install jq, which might take a few minutes.
    2. Koding.com users should download the script file below (get_jq.sh), in cygwin move to the directory where jq is downloaded, and install with the following three commands: 1) 'wget http://humanities.uva.nl/~mkoolen1/teaching/data/FOtD/2015/scripts/get_jq.sh', 2) 'chmod u+x get_jq.sh' 3) 'sudo ./get_jq.sh'. This will then download and install jq, which might take a few minutes.
      get_jq.sh
    3. Unix/Linux/Mac users can download a working 'binaries' version for their system from the jq website (OS X for Mac and Linux for Unix/Linux users). Move to the directory where you downloaded jq, change the filename (e.g. jq-osx-amd64 or jq-linux64) to just 'jq' (e.g. type 'mv jq-linux64 jq', and move the file to the folder /usr/local/bin with the command 'sudo mv jq /usr/local/bin'. The latter will require you to type your administrator password.
      jq download page
    4. All users: on the command line, type 'jq' and press enter to check if jq is installed properly. If it works, you should see on your screen information on how to use jq.

    Learn to harvest data from online sources

    The Web is filled with interesting data for humanities research. Many institutions and companies provide public access to data through APIs and search services. The latter are fine for consulting search results one at a time, but if you want to analyse many items as a whole, APIs offer you an option to download multiple items in one or several steps. In this exercise you learn to gather data from external sources through command line tools.

    Steps:

    1. The tutorial can be found here:

    Using RStudio for geocoding and mapping geopraphical data

    R is a programming language for doing statistical data analysis. Seemingly far removed from the Humanities, but it is actually used in many Digital Humanities projects. Like the Unix command line tools and a programming language like Python, it offers many options for acquiring, selecting, filtering and visualizing data. Below are some tutorial to get you acquainted with R through the RStudio environment, with examples of textual and literary analysis, movie actor network analysis and geomapping and geocoding.

    Goals:

    Steps:

    1. Download the R script below, start RStudio and open the script via 'File', 'New File', 'R script' and go through the steps in the script. Lines starting with '#' are comment lines that contain explanation of the script. You can execute each line with instructions in the script by placing the cursor on that line and clicking the 'Run' button on the top right corner of the script panel.
      Using R to map NY Times reports on student protests
    2. The following tutorial explains how to do text analysis on Shakespeare
      Text Mining the Complete Shakespeare
    3. Use the SPARQL plugin for R to get data from the Linked Open Data web about actors and directors in American films and generate network visualizations. Warning, this is somehwat advanced and requires the network tool Gephi next to R.
      SPARQL Package for R / Gephi ï¿½ Movie star graph visualization Tutorial

    Learn to use GitHub

    GitHub is a platform for storing repositories of documents. It's mainly used for version control of coding projects, but can be used for many other document types. It makes use of git, a distributed version control system that allows multiple users to share and synchronise their local copies of repositories. This allows easy collaboration and a reliable way of managing project documents such as scripts and documentation.

    Goals:

    Steps:

    1. Follow the GitHub tutorial of the Software Carpentry workshops to learn the basics.
